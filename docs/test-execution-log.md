## 四、失败原因分析

### 4.1 核心问题：login 辅助函数超时

**问题描述**：11个失败用例中，有8个与 login 函数直接相关

**失败模式**：
```
Error: Test timeout of 30000ms exceeded.
  at login() -> page.waitForURL() 或 page.goto()
```

**根本原因分析**：
1. 登录成功后页面未按预期跳转
2. `page.waitForURL(/\/$/, { timeout: 15000 })` 等待失败
3. 降级逻辑中 `page.goto(BASE_URL)` 也超时
4. 可能是应用服务器响应慢或认证状态未正确建立

### 4.2 次要问题：元素定位超时

**问题描述**：内容创建测试中定位元素超时

**失败案例**：
- TC-014: `getByRole('button', { name: '保存草稿' })` 滚动到视图超时
- TC-015: `waitForURL` 超时

**可能原因**：
1. 选择器不准确
2. 页面渲染延迟
3. 前端验证导致提交失败

### 4.3 其他问题

1. **测试数据依赖**：多个测试用例因为"无内容卡片"而跳过
2. **并行执行冲突**：4个worker可能导致数据库竞争
