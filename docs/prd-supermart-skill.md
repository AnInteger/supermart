## 1. 产品概述

### 1.1 产品愿景

SuperMart 是一个面向专业从业者的 UGC 平台，让各领域的专家能够将工作流程转换为 Skill 工作流，实现行业知识的规模化复用。

### 1.2 核心价值主张

> **让专业从业者分享实战经验，让行业知识规模化复用。**

### 1.3 目标

| 目标类型 | 描述 | 成功指标 |
|---------|------|---------|
| 用户增长 | 吸引专业从业者注册并贡献内容 | 月活用户数 (MAU) > 1000 |
| 内容生态 | 建立高质量的 Skill 内容库 | Skill 总数 > 500 |
| 用户活跃 | 促进用户互动与内容消费 | 日均下载量 > 100 |

### 1.4 产品范围

**本期范围 (MVP)**：
- Skill 完整生命周期管理（上传、浏览、下载、版本控制）
- AI 辅助功能（简介生成、安全性报告）
- 用户体系（注册、登录、个人中心）
- 互动功能（评论、评分、收藏）
- 数据统计（浏览量、下载量、收藏数）

**未来迭代**：
- Agent Bot 功能
- 付费 Skill 与收益系统
- 更多垂直领域分类

## 2. 用户画像与场景

### 2.1 目标用户

#### 用户画像 A：Skill 创作者

| 属性 | 描述 |
|-----|------|
| **角色** | 某垂直领域的专业从业者 |
| **年龄** | 25-45 岁 |
| **技能水平** | 在特定领域有 3+ 年经验 |
| **动机** | 分享专业知识，建立个人品牌，获取反馈 |
| **痛点** | 缺乏专业的知识分享平台，工作流难以标准化复用 |

**典型场景**：
> 作为一名资深前端工程师，小王开发了一套高效的代码审查 Skill。他希望通过平台分享这个 Skill，让更多开发者受益，同时获得社区反馈来改进工作流。

#### 用户画像 B：Skill 使用者

| 属性 | 描述 |
|-----|------|
| **角色** | 寻求效率提升的从业者 |
| **年龄** | 22-40 岁 |
| **技能水平** | 初中级从业者，希望学习最佳实践 |
| **动机** | 学习行业最佳实践，提升工作效率 |
| **痛点** | 缺乏高质量、可直接复用的工作流模板 |

**典型场景**：

> **场景 1：PRD 编写学习**
> 作为一名刚入行的产品经理，小李需要学习如何撰写 PRD。她在 SuperMart 上搜索到一份资深 PM 分享的 PRD 编写 Skill，下载后直接应用于工作中。

> **场景 2：视频字幕翻译**
> 作为一名内容创作者，小张需要将视频的外挂字幕翻译成多种语言。他在 SuperMart 上找到了一个"字幕批量翻译" Skill，配合 Claude 使用，快速完成了 100 个视频的字幕翻译工作。

> **场景 3：代码审查自动化**
> 作为一名团队 Tech Lead，小陈需要定期审查团队成员的代码。他下载了一个"代码审查专家" Skill，自动按照团队规范生成审查报告，大大提升了代码评审效率。

> **场景 4：竞品分析报告**
> 作为一名市场分析师，小王每周需要产出竞品分析报告。她使用平台上的"竞品分析助手" Skill，输入竞品网址后自动生成结构化的分析报告。

> **场景 5：设计规范文档生成**
> 作为一名 UI 设计师，小林需要将 Figma 设计稿转换为开发可用的设计规范文档。她找到了"设计规范导出" Skill，一键生成了包含颜色、字体、间距的完整规范文档。

#### 用户画像 C：访客

| 属性 | 描述 |
|-----|------|
| **角色** | 未登录的潜在用户 |
| **动机** | 浏览平台内容，评估是否注册 |
| **权限** | 可浏览、可下载，不可互动 |

## 3. 功能需求列表

### 3.1 Skill 管理

#### F1.1 Skill 上传

| 字段 | 内容 |
|-----|------|
| **用户故事** | 作为创作者，我希望能够上传 Skill 文件包，以便分享我的工作流 |
| **验收标准** | 1. 支持上传 ZIP 格式的 Skill 文件包<br>2. 必填字段：名称、分类、版本号、版本说明<br>3. 上传成功后自动解析目录结构<br>4. 上传后触发 AI 简介生成和安全性报告 |
| **优先级** | P0 |

#### F1.2 AI 自动生成简介

| 字段 | 内容 |
|-----|------|
| **用户故事** | 作为创作者，我希望系统自动生成 Skill 简介，以便节省时间并保持一致性 |
| **验收标准** | 生成的简介包含：<br>1. Skill 的核心功能描述<br>2. Skill 的适用场景<br>3. 一个真实案例的使用示例<br>4. 一个合理性结论 |
| **优先级** | P0 |
| **参考** | Claude Skill 定义 |

#### F1.3 安全性报告生成

| 字段 | 内容 |
|-----|------|
| **用户故事** | 作为平台，我希望自动检测 Skill 的安全性，以便保护用户免受恶意代码侵害 |
| **验收标准** | 上传时自动生成安全性报告，报告包含：<br>1. **安全性结论**：整体安全评级（安全/低风险/高风险）<br>2. **潜在风险列表**：识别出的具体安全性风险点及说明<br> |
| **优先级** | P1 |
| **参考** | [ClawHub Security Prompt](https://github.com/openclaw/clawhub/blob/main/convex/lib/securityPrompt.ts) |

#### F1.4 Skill 分类

| 字段 | 内容 |
|-----|------|
| **用户故事** | 作为使用者，我希望 Skill 按类型分类，以便快速找到所需内容 |
| **验收标准** | 初始分类：<br>1. 创作运营<br>2. 代码开发<br>3. 平面设计<br>4. 可扩展更多垂直领域 |
| **优先级** | P0 |

#### F1.5 发布新版本

| 字段 | 内容 |
|-----|------|
| **用户故事** | 作为创作者，我希望为已发布的 Skill 发布新版本，以便持续改进我的工作流 |
| **入口** | 1. 个人中心 → 我的 Skill → 发布新版本<br>2. Skill 详情页（仅作者可见）→ 发布新版本 |
| **验收标准** | 1. **已发布的 Skill 不可编辑**，只能发布新版本<br>2. 只有 Skill 作者可以发布新版本<br>3. 必须上传新的 ZIP 文件<br>4. 必须填写版本号（不能与已有版本重复）<br>5. 必须填写版本说明（描述本版本更新内容）<br>6. AI 重新生成简介和安全性报告<br>7. 发布成功后，新版本成为"当前版本" |
| **优先级** | P0 |

#### F1.6 版本历史

| 字段 | 内容 |
|-----|------|
| **用户故事** | 作为使用者，我希望查看 Skill 的版本历史，以便了解更新内容或下载旧版本 |
| **验收标准** | 1. 详情页显示"版本历史"模块<br>2. 版本列表按时间倒序排列（最新在前）<br>3. 每个版本显示：版本号、版本说明、发布时间、下载量<br>4. 可下载任意历史版本<br>5. 当前版本有特殊标识（如"最新"标签） |
| **优先级** | P1 |

#### F1.7 Skill 详情页

| 字段 | 内容 |
|-----|------|
| **用户故事** | 作为使用者，我希望查看 Skill 的完整信息，以便判断是否满足需求 |
| **验收标准** | Skill 详情页展示以下信息：<br>**基础信息**：名称、简介（AI生成）、分类、版权（MIT-0）<br>**版本信息**：当前版本号、版本说明、发布时间<br>**统计信息**：浏览量、下载量、收藏数、平均评分<br>**内容展示**：目录结构（树形）、文件预览（支持代码高亮）<br>**版本历史**：历史版本列表，可下载任意版本<br>**互动区域**：评论列表、评分入口、收藏按钮、下载按钮<br>**作者操作**（仅作者可见）：发布新版本入口 |
| **优先级** | P0 |

#### F1.8 版权声明

| 字段 | 内容 |
|-----|------|
| **用户故事** | 作为平台，我希望所有 Skill 都有明确的版权声明，以便用户了解使用权限 |
| **验收标准** | 1. 所有 Skill 默认使用 MIT-0 许可证<br>2. 在 Skill 详情页显著位置展示版权声明<br>3. 下载时提示用户版权信息 |
| **优先级** | P0 |

## 5. 非功能需求

### 5.1 性能要求

| 指标 | 要求 |
|-----|------|
| 页面加载时间 | < 2s (P95) |

| 要求 | 描述 |
|-----|------|
| 用户认证 | 使用 NextAuth v5，支持 OAuth 和邮箱登录 |

| 类型 | 要求 |

- 分类系统支持动态扩展
- 支持未来添加 Agent Bot 类型
- 数据库设计预留付费功能字段

## 6. 边界情况与异常处理

### 6.1 Skill 上传

| 场景 | 处理方式 |
|-----|---------|
| 文件格式不支持 | 提示"仅支持 ZIP 格式" |
| 文件过大（> 10MB） | 提示"文件大小不能超过 10MB" |
| 文件损坏/解压失败 | 提示"文件解压失败，请检查文件完整性" |
| Skill 名称重复 | 提示"该名称已被使用，请更换" |
| 必填字段为空 | 表单校验，高亮显示必填项 |
| 安全性检测高风险 | 阻止上传，展示风险详情 |

### 6.2 用户操作

| 场景 | 处理方式 |
|-----|---------|
| 未登录访问需权限功能 | 弹出登录框或跳转登录页，登录后返回原页面 |
| 第三方登录取消 | 用户取消授权，返回登录页，无错误提示 |
| 第三方登录失败 | 提示"登录失败，请重试或使用其他方式登录" |
| 第三方账号未绑定邮箱 | 引导用户补充邮箱信息 |
| 重复评分 | 每个用户对每个 Skill 仅能评分一次，重复操作提示"已评分" |
| 重复收藏 | 已收藏状态点击则取消收藏 |
| 评论为空 | 提交时校验，提示"评论内容不能为空" |
| 编辑他人 Skill | 403 无权限，提示"只能编辑自己的 Skill" |

### 6.3 数据边界

| 场景 | 处理方式 |
|-----|---------|
| Skill 名称过长 | 限制 100 字符，超出截断提示 |
| 评论过长 | 限制 2000 字符，实时显示剩余字数 |
| 版本号格式 | 由用户自由定义，不做强制格式校验 |
| 空列表（无 Skill） | 首页显示"暂无内容，成为第一个上传者"引导 |
| Skill 已删除 | 访问时显示"该 Skill 已被删除" |

### 6.4 系统异常

| 场景 | 处理方式 |
|-----|---------|
| AI 简介生成失败 | 允许用户手动编辑简介，后台异步重试 |
| 网络请求超时 | 提示"网络异常，请稍后重试"，支持重试 |
| 服务不可用 | 友好错误页面，显示状态信息 |
| 文件下载失败 | 提示"下载失败，请重试" |
