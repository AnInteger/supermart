## 一、产品概述

**SuperMart - 国内垂类AI工具平台**

一个面向专业从业者的UGC平台，让软件开发、建筑、法律、设计、社媒运营等行业的专家能够：
- 将工作流程转换为Skill工作流
- 将领域知识转换为Agent Bot
- 通过AI辅助提升工作效率

平台采用混合模式：基础内容免费共享，高级内容后期引入付费机制，同时支持企业私有部署。初期MVP阶段专注免费内容，快速建立生态。

**核心价值主张**：让专业从业者分享实战经验，让行业知识规模化复用。

## 二、Skill/Agent 定义与数据模型

### 2.1 什么是 Skill？

**Skill 是 AI Agent 的可复用能力单元**，包含：
- 结构化的指令（告诉 AI 如何执行特定任务）
- 必要的上下文知识
- 可选的工具、脚本和模板

**参考定义**：
- [AgentSkills.io](https://agentskills.io/what-are-skills)：Skill 是包含 `SKILL.md` 文件的文件夹
- [Claude Code Skills](https://claude.com/resources/tutorials/what-are-skills)：扩展 Claude 能力的专业化知识和工作流

### 2.2 什么是 Agent？

**Agent 是具有自主执行能力的 AI 实体**，可以：
- 理解用户意图
- 调用多个 Skill 完成复杂任务
- 与外部工具/API 交互
- 独立做出决策

### 2.3 好的 Skill 是什么样的？

一个高质量的 Skill 应该具备：

| 维度 | 标准 | 示例 |
|------|------|------|
| **清晰的目标** | 一句话说明解决什么问题 | "帮助开发者进行代码审查" |
| **明确的触发条件** | 用户何时应该使用这个 Skill | "当用户请求代码审查时" |
| **完整的指令** | 步骤清晰，可执行 | 分步骤的操作指南 |
| **必要的上下文** | 包含所需知识，不过度依赖外部 | 内置代码规范、最佳实践 |
| **工具配置** | 明确需要哪些工具/API | ESLint、Prettier 配置 |
| **使用示例** | 至少 1-2 个真实案例 | 前后对比、输入输出示例 |
| **边界说明** | 能做什么，不能做什么 | "不适用于大型重构" |

**反例（差的 Skill）**：
- 描述模糊："帮助用户更好地工作"
- 没有具体步骤
- 缺少必要的配置信息
- 没有示例

## 三、目标用户与商业模式

### 3.1 目标用户

**主要用户：专业从业者**
- 软件开发工程师、架构师
- 建筑设计师、结构工程师
- 律师、法务人员
- UI/UX设计师、平面设计师
- 国内社媒运营人员
- 其他垂直行业专业人士

**用户角色**：
- **普通用户**：浏览、使用、发布内容（UGC核心）
- **运维人员**：后台管理、内容审核、私有内容管理

### 3.2 商业模式

**发展阶段**：
1. **MVP阶段**（1-2个月）：专注免费内容，建立生态
2. **成长阶段**（3-6个月）：引入付费内容（认证专家、高级内容）
3. **成熟阶段**（6个月+）：企业版私有部署、订阅制

**收入来源**（后期）：
- 付费内容抽成（20-30%）
- 企业版订阅
- API调用收费

## 四、平台功能架构

### 4.1 平台形态
- **Web应用**：浏览器访问，跨平台兼容，PWA支持移动端体验

### 4.2 内容创建方式
1. **模板化创建**（后期）：提供行业模板，用户填写配置即可快速上手
2. **AI辅助生成**（后期）：用户描述需求，AI自动生成Skill/Agent原型
3. **导入导出**（后期）：支持标准格式导入导出，对接Claude Code、LangChain等生态
4. **手动创建**（MVP）：支持文本+JSON配置的手动填写

### 4.3 内容质量控制

**MVP阶段**：
- **社区自治**：用户评分、评论机制（唯一质量控制手段）

**后期增强**：
- 专家认证：专业从业者身份认证
- 使用数据驱动：根据使用次数、成功率自动推荐
- 付费门槛：自然筛选质量

## 五、MVP功能范围

### 5.1 核心功能（必须包含）

#### 1. 用户管理
- 邮箱/手机号注册
- 密码登录
- 基础用户信息管理
- 我创建的内容
- 我收藏的内容
- 个人资料编辑

#### 2. Skill/Agent浏览和搜索
- 分类浏览（按行业、类型）
- 关键词搜索
- 筛选排序（最新、最热、评分最高）

#### 3. 内容详情页
- Skill/Agent基本信息（名称、描述、作者）
- 操作手册（使用说明）
- 工具配置（需要的工具和API）
- 分步设置指南
- 使用案例/示例

#### 4. 评论和评分
- 五星评分系统
- 文字评论

#### 5. 基础内容创建（新增）
- 文本编辑器（Markdown支持）
- JSON配置编辑器
- 文件上传（脚本、模板等）
- 预览功能
- 发布/保存草稿

### 5.2 后期功能（暂不包含在MVP）
- 内容创建工具（模板化/AI辅助）
- 后台管理系统
- 导入导出功能
- 付费内容
- 专家认证

## 七、技术架构

### 7.1 架构选型理由

**选择全栈单体架构**：
- MVP阶段需要快速迭代验证
- AI调用主要是HTTP请求，Node.js完全胜任
- 单一代码库降低心智负担和部署复杂度
- 后期可按需拆分出独立服务

```
┌─────────────────────────────────────────────┐
│              Next.js 15 全栈应用             │
│  ┌─────────────┐  ┌─────────────────────┐   │
│  │   前端UI    │  │   Server Actions    │   │
│  │  (React)    │  │   (API逻辑)         │   │
│  └─────────────┘  └─────────────────────┘   │
│                    ┌─────────────────────┐   │
│                    │   Prisma ORM        │   │
│                    └──────────┬──────────┘   │
└───────────────────────────────┼──────────────┘
                                │
              ┌─────────────────┼─────────────────┐
              ▼                 ▼                 ▼
        ┌──────────┐      ┌──────────┐     ┌──────────┐
        │PostgreSQL│      │  Redis   │     │ 对象存储  │
        │ (主数据库)│      │ (缓存)   │     │ (文件)   │
        └──────────┘      └──────────┘     └──────────┘
```

### 7.2 技术栈详情

| 层级 | 技术 | 选型理由 |
|------|------|----------|
| **框架** | Next.js 15 (App Router) | 全栈能力、SSR/SSG原生支持、SEO友好 |
| **语言** | TypeScript | 类型安全、减少运行时错误 |
| **UI组件** | Tailwind CSS + shadcn/ui | 开发效率高、样式一致、可定制 |
| **ORM** | Prisma | 类型安全、迁移管理方便、与TS集成好 |
| **数据库** | PostgreSQL | 可靠稳定、Prisma支持完善、免费 |
| **缓存** | Redis (Upstash) | 搜索缓存、会话存储、免费额度够用 |
| **认证** | Auth.js (NextAuth) | 成熟方案、支持多种登录方式 |
| **文件存储** | Cloudflare R2 | S3兼容、免费额度大、无出口流量费 |
| **部署** | Vercel | Next.js原生支持、免费额度、自动CI/CD |
| **搜索** | PostgreSQL全文搜索 | MVP够用、后期可升级向量搜索 |

### 7.3 AI模型对接

**MVP阶段**：
- 对接 1-2 个国产大模型（如通义千问、智谱GLM）
- 用户可配置自己的API Key
- 通过 Server Actions 调用模型API

**后期扩展**：
- 文心一言、DeepSeek等
- 模型适配层统一接口
- 向量数据库支持语义搜索（Pinecone/Milvus）

### 7.4 部署架构

```
┌─────────────────────────────────────────────────┐
│                    Vercel                        │
│  ┌───────────────────────────────────────────┐  │
│  │         Next.js 应用 (Edge/Serverless)    │  │
│  └───────────────────────────────────────────┘  │
└─────────────────────────────────────────────────┘
          │              │              │
          ▼              ▼              ▼
    ┌──────────┐   ┌──────────┐   ┌──────────┐
    │Supabase  │   │ Upstash  │   │Cloudflare│
    │(Postgres)│   │ (Redis)  │   │   R2     │
    └──────────┘   └──────────┘   └──────────┘
```

**成本估算（MVP月度）**：
- Vercel: 免费额度（Hobby计划）
- Supabase: 免费（500MB数据库）
- Upstash Redis: 免费（10,000请求/天）
- Cloudflare R2: 免费（10GB存储）
- **总计：$0/月**（MVP阶段完全免费）

## 八、差异化竞争策略

### 8.1 核心差异化优势

1. **行业垂直**
   - 深度聚焦国内行业需求（软件、建筑、法律、设计、社媒运营）
   - 而非通用AI工具平台

2. **本土化**
   - 优先支持国产大模型（通义千问、文心一言、智谱GLM、DeepSeek等）
   - 符合国内合规要求

3. **专业深度**
   - 强调操作手册的完整性
   - 强调工具配置的清晰性
   - 强调分步设置指南的详细性

4. **UGC社区**
   - 让真正的专业从业者分享实战经验
   - 而非AI生成的泛化内容

5. **企业友好**
   - 支持企业私有部署（后期）
   - 支持团队管理（后期）

6. **AI辅助创作**
   - 通过AI降低内容创建门槛（后期）
   - 让非技术用户也能创建复杂工作流

## 九、产品路线图

### 9.1 阶段一：MVP（1-2个月）
- 完成核心功能开发：用户管理、浏览搜索、内容详情、评论评分、基础创建
- 导入初始示例内容（60+ Skill/Agent）
- 邀请种子用户创建内容
- 验证核心假设：专业从业者愿意分享工作流程

### 9.2 阶段二：内容生态（3-6个月）
- 上线内容创建工具（模板化 + AI辅助）
- 引入专家认证机制
- 扩展更多行业覆盖
- 优化搜索推荐算法
- 引入向量数据库支持语义搜索

### 9.3 阶段三：商业化（6个月+）
- 引入付费内容机制
- 开放API平台
- 推出企业版私有部署
- 建立分成体系
